# 单笔转账

## 功能说明

自动化完成农行企业网银的登录，自动导航至单笔转账操作页面，并根据示例供应链合同付款单据图片自动填写转账表单。

## 运行流程

1. **打开农行首页** - 访问农行官网，等待页面加载完成
2. **点击企业网银登录** - 捕获新标签页跳转，进入企业网银登录页面
3. **点击证书登录** - 等待 `#m-kbbtn-new` 按钮可见后点击（`no_wait_after=True`，不等待后续跳转）
4. **处理证书选择弹窗** - 发送 1 次 Enter 键确认证书选择（当前使用 Enter 键兜底策略）
5. **输入K宝密码** - 从 `.env` 读取密码，等待 2 秒让密码框获得焦点后逐位输入（每位间隔 0.08s），最后回车提交
6. **导航到单笔转账** - 等待页面加载完成后，依次点击顶部导航「付款业务」-> 子菜单「单笔转账」
7. **读取单据并填写表单** - 读取示例供应链合同付款单据图片，自动提取数据填入表单（需配置 TRANSFER_IMAGE_PATH）
8. **截图保存** - 截图保存至 `skills/单笔转账/screenshot.png`
9. **保持浏览器运行** - 浏览器保持运行，按 Ctrl+C 退出并关闭浏览器

## 字段映射

示例供应链单据字段与农行表单的对应关系：

| 示例供应链字段 | 农行表单字段 | 说明 |
|-----------|------------|------|
| 银行账户 | 收款账号 | 收款方银行账号 |
| 收款单位名称 | 收款户名 | 收款方单位/个人名称 |
| 开户银行 | 收款方开户行 | 收款方开户银行全称 |
| 申请金额 | 金额 | 转账金额 |
| 申请说明 | 用途 | 转账用途说明 |

## 已实现但未启用的函数

以下函数已在代码中定义，但当前 `main()` 未调用：

| 函数名 | 功能 | 说明 |
|-------|------|------|
| `close_tip_dialog_if_needed()` | 关闭"温馨提示"弹窗 | 检测 `.el-dialog__wrapper` / `.el-message-box__wrapper` 中包含"温馨提示"文本的弹窗并关闭 |
| `click_confirm_in_certificate_dialog()` | DOM/JS方式点击证书弹窗确定按钮 | 三层策略：DOM选择器 -> JS evaluate -> 返回 False 触发 Enter 兜底 |
| `fallback_press_enter()` | Enter键多次兜底 | 支持自定义次数和间隔，默认12次每次0.6s |

## 环境配置

在项目根目录的 `.env` 文件中配置：

```
KB_PASSWORD=你的K宝密码
TRANSFER_IMAGE_PATH=示例供应链合同付款单据图片路径（如 C:/path/to/contract.png）
```

> 注意：不配置 `TRANSFER_IMAGE_PATH` 时会跳过表单自动填写步骤，仅登录导航到单笔转账页面。
