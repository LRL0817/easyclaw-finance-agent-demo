# 日记账 Skill

## 用途
- 复用项目现有登录能力，不重复实现账号登录逻辑。
- 通过同目录 `日记账.py` 调用项目根目录 `ubank_common.py` 的公共函数完成自动化流程。
- 支持两种运行模式：**查询模式**（完整U-BANK自动化）和**写入模式**（仅JSON写入Excel）。

## 两种模式

### 模式1：查询模式（默认）
启动U-BANK，自动登录，进入交易明细页面，提取数据，保存JSON并追加写入Excel。

```bash
python "skills/日记账skill - 悠动类型/日记账.py"
```

流程：打开U-BANK -> 登录 -> 进入交易明细 -> 设置日期范围 -> 查询 -> 提取数据 -> 保存JSON -> 追加写入Excel -> 关闭退出

### 模式2：写入模式
从已有JSON文件读取交易数据，追加写入Excel文件（不启动U-BANK）。

```bash
# 使用最新的JSON文件
python "skills/日记账skill - 悠动类型/日记账.py" -w

# 指定具体的JSON文件
python "skills/日记账skill - 悠动类型/日记账.py" -w "screenshots/rijizhang_data_20260422_175229.json"
```

去重逻辑：用 `(日期 + 收入 + 支出 + 余额)` 四元组判断，已存在的记录会跳过，只追加新记录。只操作A~G列，H/I列不动。

## 执行步骤
1. 确认在项目根目录运行。
2. 根据需要选择模式执行命令。
3. 观察终端输出，确认数据提取和写入结果。

## 关键文件
- `日记账.py`：日记账入口脚本（复用公共登录与退出模块）
- `ubank_common.py`：登录、窗口等待、控件点击、退出确认等公共能力（项目根目录）
- `4--北京示例付款单位B-银行日记账.xlsx`：目标Excel文件（数据追加写入此文件）

## 输出文件
- JSON数据：`screenshots/rijizhang_data_时间戳.json`
- Excel数据：直接追加到 `4--北京示例付款单位B-银行日记账.xlsx`
