# 单笔转账 Skill

自动化兴业银行企业网银的登录流程，为后续的单笔转账操作打好基础。

## 运行

```bash
python "兴业银行\skills\单笔转账\open_bank.py"
```

## 依赖

- Python：`pyautogui`, `pygetwindow`, `pywin32`
- 文件：
- 快捷方式默认读取当前用户桌面 `兴业银行企业网银.lnk`，也可用 `CIB_SHORTCUT_PATH` 覆盖
- 配置 `兴业银行\.env`，包含：
    - `LOGIN_PWD`：网盾（UKey）密码
    - `CERT_PWD`：登录页证书密码
- 截图输出目录 `兴业银行\screenshots`

## 流程

1. **打开APP**：若"兴业"窗口未开，则 `os.startfile` 快捷方式，等待窗口出现（最多30秒）。
2. **网盾密码**：打开APP后对话框默认聚焦，**直接用物理键盘** `pyautogui.write(LOGIN_PWD)` 敲入，再 `press("enter")` 确认。不做窗口检测、不点击输入框——绕过软键盘随机化，也规避内嵌子窗口无法被 `pygetwindow` 枚举的问题。
3. **登录页**（基于 1920×1080 的屏幕坐标）：
   - 下拉框 `(1331, 395)` → 选第一项 `(1331, 444)`（偏移 +49）
   - 密码框 `(1331, 453)` → `write(CERT_PWD)`
   - 勾选同意 `(1132, 489)`
   - 登录按钮 `(1331, 577)`
4. **登录后网盾密码**：若再次弹出对话框，调用 `handle_ukey_dialog` 重复输入 `LOGIN_PWD`。

## 关键设计决策

- **不检测网盾对话框**：该对话框是主客户端的子窗口，`pygetwindow` 的 `getAllWindows()` 只枚举顶层窗口，搜不到。改为"APP打开后直接敲键盘"，简单且可靠。
- **物理键盘输入**：兴业网盾密码框带随机软键盘，必须用 `pyautogui.write` 走系统键盘事件才能正确输入。
- **截图留痕**：每个关键步骤都落盘 PNG 到 `screenshots/`，便于事后排查坐标漂移。

## 已知约束

- 屏幕分辨率变化会使登录页坐标失效，需要基于实际截图重新测量。
- 若快捷方式启动后对话框未聚焦（极少见），网盾密码会打到错误的窗口上——此时可在第 2 步前加 `win.activate()` 或提高 `time.sleep(2)` 的等待时长。
